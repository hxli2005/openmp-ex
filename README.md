# 计算机体系结构 实验二 (Computer Architecture - Ex2)

本项目是关于 **多核并行计算与底层体系结构优化** 的实验代码库。
实验核心利用 **OpenMP指令集** 与 **C语言**，在多线程环境下探索了典型的并行编程模式、底层的性能瓶颈（如缓存一致性风暴），并通过多种优化手段（TLP、SIMD、ILP）挖掘现代 CPU 的极限算力。

## 📁 目录结构与模块说明

### 1. `pi/` (核心基准测试：圆周率计算)
基于定积分法计算 $\pi$ 的演进式性能优化实验。包含6种不同架构策略的详尽对比如下：
- **串行基准 (Serial)**：单核基准线。
- **并行方法 1 (交替迭代 + 共享数组)**：展示了极其典型的性能杀手 **伪共享 (False Sharing)** 现象。
- **并行方法 2 (静态分块 + 局部数组)**：展示连续访问模式对高速缓存预取友好性带来的性能回升。
- **并行方法 3 (临界区 + 私有变量)**：利用核心私有栈和 `#pragma omp critical`，隔离多线程修改冲突。
- **并行方法 4 (规约 Reduction)**：OpenMP 工业级标准实现，运用自动树状归约消除同步开销。
- **并行方法 5 (多线程 + SIMD)**：线程级并行(TLP)与向量化数据级并行(SIMD)的极致结合。
- **并行方法 6 (指令级并行 ILP 展开)**：进一步配合激进编译优化(`-O3 -ffast-math`)，通过多路无依赖累加解决浮点模块的真数据依赖冒险(Data Hazard)。

### 2. `others/` (体系结构现象剥析微实验)
包含多个针对特定并发陷阱和原理的极简验证程序：
- ` private.c`：探讨 **共享变量 (Shared)** 与 **私有变量 (Private)** 在线程栈上的内存生命周期和隔离机制。
- `burden.c`：证明**并行额外负担 (Overhead)**，演示在极小规模任务下开启多线程不仅无法加速，反而被系统调度和同步屏障严重点累的现象（粒度控制）。
- `load.c`：演示 **线程负载不均衡 (Load Imbalance)**，对比 `schedule(static)` 与工作窃取 `schedule(dynamic)` 的耗时区别。
- `sync.c`：复现多核争抢写入同一变量时的**数据竞争 (Data Race)**，并对比评价 `critical` 锁、`atomic` 硬件级原子指令、与 `barrier` 时间点屏障的开销。

### 3. `matrix/` (矩阵乘法)
针对计算密集型任务（矩阵乘法）在多线程分块处理、SIMD向量化（如 AVX-512）方面的扩展优化实验及其基准对比 (`*.csv`)。

### 4. `helllo _world/`
OpenMP 并行环境的探针与环境测试用例。

## 🛠️ 编译与运行说明 (针对 macOS 环境)
因 macOS 原生 Clang 默认不支持 OpenMP，需借助 Homebrew 环境下的 llvm / libomp。
典型编译命令模板：
```bash
clang -Xpreprocessor -fopenmp -I$(brew --prefix libomp)/include -L$(brew --prefix libomp)/lib -lomp -O3 -ffast-math 源文件.c -o 目标文件
./目标文件
```
*(注意：在 `pi` 目录与 `matrix` 目录下，实验通过各自内部的框架进行了更复杂的迭代时长测评，直接运行相应生成的可执行程序即可打出完整的比较表格)。*
